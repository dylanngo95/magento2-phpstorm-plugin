Action classes are extensions of the Action class that a router returns on matched requests. Each Action should implement Magento\Framework\App\Action\HttpHTTP MethodActionInterface to declare which HTTP request methods it can process. The execute() function in these classes contain the logic for dispatching requests. Action class should return a result object. Backend Controller allows to restrict an access by ACL.
Read more about Controller Action, including when those can be used.
Predefined variables will take the following values:
${NAMESPACE}   Created PHP Controller Action class namespace.
${NAME}   PHP class for the Controller Action name.
${EXTENDS}   Name of PHP class that the Controller Action extends.
${IMPLEMENTS}   Name of PHP interface that the Controller Action implements.
${USES}   List of imports separated by comma.
${ACL}   The name of ACL resource that allows to restrict an access to the backend controller action.
